1. The flow

python main.py --customer <id> runs one linear path:

main.py:main()
 └─ tools/bank_report.py:generate_bank_report(cid, theme)
     ├─ 1. pipeline/reports/customer_report_builder.py:build_customer_report(cid)   ← deterministic build
     │      ├─ data/loader.py:get_transactions_df()          (rgs.csv, cached in module global)
     │      ├─ tools/analytics.py:get_spending_by_category, get_cash_flow
     │      ├─ tools/transaction_fetcher.py:fetch_transaction_summary  (called TWICE)
     │      ├─ tools/category/resolver.py:resolve_category_presence    (rent, utilities)
     │      ├─ tools/merchant_features.py:compute_all_merchant_features (13 sub-features)
     │      ├─ tools/account_quality.py:compute_account_quality        (loads rg_sal CSV itself)
     │      ├─ tools/event_detector.py:detect_events                   (loads rg_sal CSV itself)
     │      └─ pipeline/reports/checklist_builder.py:compute_checklist
     ├─ 2. data/loader.py:load_rg_salary_data(cid)            (rg_sal/rg_income CSVs — 3rd read)
     ├─ 3. pipeline/reports/report_summary_chain.py:generate_customer_review   ← the ONLY LLM call
     │      └─ utils/llm_factory.create_chat_model + config/prompts.CUSTOMER_REVIEW_PROMPT
     ├─ 4. pipeline/renderers/combined_report_renderer.py:render_combined_report
     │      ├─ tools/scorecard.py:compute_scorecard            (data computation inside the renderer)
     │      ├─ pipeline/renderers/bank_v2_view_model.py:build_bank_v2_context
     │      │      └─ re-loads transactions + tools/analytics.get_balance_trend
     │      └─ Jinja render of templates/bank_report_v2.html   (written twice: output + copy dir)
     └─ 5. tools/excel_exporter.py:build_excel_row + export_row_to_excel
 └─ (optional --serve) pipeline/renderers/serve_report.py:serve
